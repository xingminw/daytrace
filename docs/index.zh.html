AI 工具替你留下的痕迹，反过来讲清你这一天。

你现在真正干活的地方都在 Codex、Claude Code、Cursor、DeepSeek 这些工具里。 它们每天替你产生海量痕迹，但几乎没有系统在替你把它们收起来。
DayTrace 干的就是这件事：本地优先、数据只在你机器上， 把 AI 工具、git、文档每天产生的信号都收回来，对齐你真实的任务， 最后还你一份 你自己愿意读 的日报。

你直接能看到的

一眼看完的 4 块
事件数 / 活跃时长 / 最长专注块 / AI 花费 —— 进页面 1 秒掌握今天大致样子。
用你自己口气写的日报
不是机器人的"今日完成 X 件事"，而是顺着你真实做过的事，像同事讲给你听。
Insights 不来虚的
任务推进、今日 vs 7 日基线、临近 deadline 提醒。没有"今天要更专注哦"那种假关心。
图表和 dashboard 同一套配色
按任务堆叠的柱状图、总量 donut，直接嵌进邮件和飞书云文档里。
周回顾 = 7 张日卡片
一周横着摊开，七天各是一段故事，不是被压扁的一行表。
整个系统就一个 SQLite 文件
所有记录都在那里。没有云、没有埋点、没有第三方碰你的数据。

快速上手

# clone、装依赖、指向你的数据。
git clone https://github.com/xingminw/daytrace
cd daytrace
make install

# 配 DeepSeek + Gmail（可选，但日报里 AI 那块要靠它）。
mkdir -p ~/.daytrace
cat > ~/.daytrace/secrets.env <<'EOF'
DEEPSEEK_API_KEY=sk-...
DAYTRACE_GMAIL_USER=
DAYTRACE_GMAIL_APP_PASSWORD=
DAYTRACE_EMAIL_TO=
EOF
chmod 600 ~/.daytrace/secrets.env

# 跑一下。
make daily        # 采集 + 给昨天写总结
make dashboard    # 打开 http://127.0.0.1:8765/today
